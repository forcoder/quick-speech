# 智能输入法APP设计文档

**日期**: 2026-04-30
**版本**: v1.0
**定位**: 面向企业/办公人群的智能输入法，AI智能回复为核心卖点

---

## 1. 产品定位

一款基于安卓平台的智能输入法应用，以五笔输入为基础，以「AI智能回复」为核心差异化功能。通过云端知识库和AI智能体，帮助企业用户在邮件、即时通讯、文档等办公场景中快速生成高质量回复。

## 2. 目标用户

- **主要**: 企业办公人员（客服、销售、运营、管理）
- **次要**: 需要高效文字输入和回复的专业人士
- **管理员**: 企业IT/知识管理员（负责知识库和AI智能体配置）

## 3. 技术选型

| 模块 | 技术栈 |
|------|--------|
| 安卓客户端 | Kotlin + Jetpack Compose |
| 五笔引擎 | 自研C++引擎 + JNI调用 |
| 后端API | Spring Boot + PostgreSQL + Redis |
| 向量检索 | Milvus（知识库向量化） |
| AI集成 | OpenAI / Claude / 国内大模型API（可配置） |
| 管理后台 | React + TypeScript + Ant Design |
| 部署 | Docker + Kubernetes |

## 5. 核心功能模块

### 5.1 五笔输入引擎
- 支持86/98五笔方案
- 智能纠错与容错输入
- 云词库同步与个性化词频学习
- 英文/数字/符号混合输入

### 5.2 AI智能回复（核心）

#### 双模式设计
| 模式 | 说明 | 适用场景 |
|------|------|----------|
| 📚 知识库模式（RAG） | 检索知识库后生成回复 | 有知识沉淀的FAQ、产品文档、话术 |
| 🤖 AI智能体模式 | 调用配置的AI智能体生成 | 创意生成、复杂推理、多轮对话 |
| 🔀 混合模式（默认） | 同时查询两种模式，综合排序展示 | 通用场景，给用户最优结果 |

#### 模式切换
- 默认混合模式，同时查询知识库和AI智能体
- 每条回复标注来源标签（📚/🤖/🔀）
- 用户可通过面板按钮手动切换到纯知识库或纯AI智能体模式
- 用户偏好持久化保存

#### 上下文感知
- 自动识别当前应用类型（邮件客户端/IM/文档编辑器）
- 根据上下文调整回复策略（正式/非正式、长短、语气）
- 支持用户手动选择场景覆盖自动识别

### 5.3 知识库管理

#### Web管理后台
- 文档上传：支持PDF、Word、Excel、TXT、Markdown、网页URL
- 自动分块与向量化
- 知识库分类与标签管理
- 启用/禁用控制
- 更新与版本管理
- 检索测试工具

#### APP端
- 知识库列表查看（启用状态）
- 单个知识库启用/禁用切换
- 基础搜索测试

### 5.4 AI智能体配置

#### Web管理后台
- 智能体创建与管理
- Prompt模板编辑器
- 模型选择（GPT-4/Claude/国内大模型等）
- 参数调节（temperature、max_tokens等）
- 场景绑定（按APP类型自动切换）
- 测试对话窗口

#### APP端
- 智能体列表查看
- 默认智能体选择
- 场景绑定查看

### 5.5 用户系统
- 企业/团队管理
- 用户注册与登录（邮箱/手机/SSO）
- 角色权限（管理员/普通用户）
- 个人偏好设置（回复风格、常用场景）

### 5.6 数据安全
- 端到端加密传输
- 企业数据隔离（多租户架构）
- 知识库数据加密存储
- 访问日志与审计

## 6. 界面设计原则

- **简洁美观**: 遵循Material Design 3设计规范
- **高效操作**: 核心功能3步内可达
- **不干扰输入**: AI回复面板可收起，不影响正常输入
- **清晰区分**: 不同模式的结果有明确视觉区分

## 7. 数据流

### AI回复生成流程
```
1. 用户在任意APP中输入文字
2. 输入法捕获当前输入上下文 + 应用类型
3. 发送到云端API
4. 云端处理：
   a. [知识库路径] 向量检索 → 获取相关文档片段 → 构建Prompt → 生成回复
   b. [智能体路径] 加载智能体配置 → 构建Prompt → 调用AI模型 → 生成回复
   c. [混合模式] 并行执行a和b → 综合排序 → 返回Top N结果
5. 输入法展示候选回复列表（标注来源）
6. 用户点击 → 插入到输入框
```

## 11. 自进化功能（核心差异化）

### 10.1 功能概述

输入法具备持续学习能力，越用越懂用户。包含三个核心能力：
1. **知识库可编辑** — 用户可检索、查看并编辑知识库内容
2. **回复风格学习** — 系统学习用户的回复风格偏好
3. **风格化生成** — 基于用户风格生成个性化智能回复

### 10.2 知识库检索与编辑

#### 检索能力
- 在输入法面板中提供知识库搜索入口
- 支持关键词搜索和语义搜索（向量检索）
- 搜索结果高亮显示匹配片段
- 显示知识库来源文档和更新时间

#### 编辑能力
- 用户可编辑知识库检索到的内容片段
- 修改后保存为「用户修正版本」，不影响原始知识库
- 用户修正版本优先级高于原始知识库（RAG检索时优先使用）
- 支持将修正内容提交给管理员审核，通过后合并到正式知识库
- 编辑历史可追溯、可回滚

#### 数据模型
```
知识库条目
├── 原始内容（管理员维护）
├── 用户修正内容（用户维护）
│   ├── 修正文本
│   ├── 修正用户
│   ├── 修正时间
│   └── 修正原因（可选）
└── 合并状态（pending/approved/rejected）
```

### 10.3 回复风格学习

#### 学习机制
- **隐式学习**: 系统记录用户每次的行为信号
  - 用户采纳了哪些回复 → 正向样本
  - 用户拒绝了哪些回复（跳过/删除）→ 负向样本
  - 用户采纳后做了哪些修改 → 风格修正信号
  - 用户自己输入的回复 → 真实风格样本
- **显式学习**: 用户可主动配置
  - 回复风格偏好设置（正式程度、简洁程度、语气偏好）
  - 常用回复模板收藏
  - "更像这样" / "不要这样" 的反馈按钮

#### 风格模型
```
用户风格画像
├── 语言风格
│   ├── 正式程度（1-10分）
│   ├── 简洁程度（1-10分）
│   ├── 常用词汇/短语
│   ├── 句式偏好（长句/短句/混合）
│   └── 标点习惯
├── 内容风格
│   ├── 是否常用敬语
│   ├── 是否包含解释说明
│   ├── 回复长度偏好
│   └── 结构偏好（列表/段落/要点）
└── 场景风格
    ├── 邮件场景风格
    ├── IM场景风格
    └── 文档场景风格
```

#### 学习流程
```
1. 初始状态：使用默认风格模板
2. 收集阶段：记录用户前50-100次交互
3. 分析阶段：AI分析用户行为，生成风格画像
4. 应用阶段：将风格画像注入Prompt，影响回复生成
5. 持续优化：每次交互后微调风格画像
```

### 10.4 风格化生成

#### 生成策略
```
原始AI回复 → 风格转换层 → 风格化回复

风格转换层：
├── 加载用户风格画像
├── 构建风格指令（System Prompt注入）
├── 将风格指令与原始Prompt结合
└── 生成符合用户风格的回复
```

#### 实现方式
- **Prompt注入**: 将风格描述作为System Prompt的一部分
- **后处理调整**: 对AI生成的回复进行风格微调
- **Few-shot示例**: 使用用户历史回复作为示例引导生成

#### 用户控制
- 风格强度调节滑块（1-10，1=保持AI原始风格，10=完全像用户）
- 可随时关闭风格化功能
- 支持"重置风格学习"，清除所有学习数据
- 风格画像可视化展示（让用户看到系统学到的内容）

### 10.5 隐私与数据安全
- 风格学习数据本地加密存储
- 用户可选择仅本地学习或云端同步
- 随时可导出/删除个人风格数据
- 企业模式下，管理员可配置是否允许风格学习
